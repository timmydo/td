# seed/mes — GNU Mes source (source-bootstrap brick 2)

Vendored GNU Mes source, the second rung of td's from-source `/td/store` toolchain
(north star: no guix *bytes*). Brick 1 produced a minimal C compiler (M2-Planet) +
mescc-tools (M1/hex2/blood-elf) from the 229-byte stage0 seed; this tree is what those
tools compile into a working **GNU Mes Scheme interpreter** (`mes-m2`).

## Pin

- Upstream: GNU Mes — https://git.savannah.gnu.org/git/mes.git
- Commit: `f244b14191f8eecdc18eebba031a3c666fbfe28a` (VERSION 0.27.1, 2025-08-18)
- License: GPL-3.0-or-later (see `COPYING`).

This commit is chosen to **version-match** the vendored stage0-posix M2-Planet
(`seed/stage0`, pin `3b9c2bb`): its `lib/m2/x86_64/x86_64_defs.M1` uses the same
instruction-macro vocabulary the stage0 M2-Planet emits (e.g. `xor_eax,eax`). The
0.27 *release* defs are older and lack those macros, so M1 cannot assemble M2-Planet's
output against them — hence master `f244b141`, not the 0.27 tag.

## What is vendored (and what is not)

A precise subset of the upstream tree — enough to build `mes-m2` and run it, nothing
for other architectures:

- `m2planet.files` — the ordered list of C/header translation units fed to M2-Planet
  (taken from upstream `kaem.run`, `${mes_cpu}`→`x86_64`). The gate reads it verbatim.
  Its first entry `include/mes/config.h` is *generated* by the gate (see below), not
  vendored, but it must still be passed to M2-Planet in this position — so it stays in
  the manifest.
- `include/`, `lib/`, `src/` — the C sources/headers named in the manifest, plus the
  x86_64 assembly stage files `lib/m2/x86_64/x86_64_defs.M1`,
  `lib/x86_64-mes/x86_64.M1`, `lib/linux/x86_64-mes-m2/crt1.M1`,
  `lib/m2/x86_64/ELF-x86_64.hex2`, and the arch headers
  `include/linux/x86_64/{kernel-stat,signal,syscall}.h`.
- `mes/module/`, `module/` — the full Mes Scheme library trees, loaded by `mes-m2` at
  runtime (the boot chain `boot-5.scm` → core/srfi modules). Vendored whole rather than
  minimized: a missing module fails the build *loudly*, but the full tree is the clean,
  auditable "the Scheme library" boundary.

**Generated by the gate, not vendored:** `include/mes/config.h` (two lines:
`#undef SYSTEM_LIBC` / `#define MES_VERSION "0.27.1"`, mirroring upstream
`configure.sh` for the non-system-libc path) and `include/arch/*.h` (copied from
`include/linux/x86_64/`, as `configure.sh` does).

The `bootstrap-mes` gate (`mk/gates/364`, `tests/bootstrap-mes.sh`) drives the build
with guix/Guile scrubbed from the environment and asserts the result is a real Scheme
interpreter, guix-free, and reproducible.
