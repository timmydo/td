summary: Byte-reproducible MODERN toolchain at /td/store (gcc 14.3.0 + glibc 2.41 + binutils 2.44), so the content-addressed interned path is STABLE build-to-build and cross-machine — the prerequisite for [[td-subst-track]]'s repro-equality leg and consumer fetch (skip the ~90-min rebuild). Today the modern rungs ([[td-gcc-14-store-native]], [[td-glibc-241-store-native]], gate 406/410/412) intern with NO repro leg and the CA path VARIES per build ([[td-toolchain-store-native]] caveat: "cc1 stamp"). Root cause (td NAR hashes only contents/exec-bit/structure — fs mtimes are irrelevant): (1) the modern compiler/binutils binaries are built `-g` in a random mktemp dir → the absolute build path leaks into DWARF (.debug_* comp_dir); (2) installed `.a` archives carry real member mtimes from the mesboot build-ar (`--enable-deterministic-archives` only sets the BUILT ar's default, not the one used during the build). Fix: a reusable post-install normalization (`tests/repro-lib.sh repro_normalize_tree`) — `strip --strip-debug --enable-deterministic-archives` over every ELF + ar archive (keeps the symbol table so static libs/objects still link; zeros archive member mtime/uid/gid; removes the build-path-bearing debug sections). Durable repro leg per gate: build the final modern rung TWICE, normalize both, assert the interned CA path is byte-identical (intrinsic double-build reproducibility — no guix oracle). Increment 1 (this PR): binutils-2.44 (rung A, the cheapest modern rung) reproducible end-to-end + the shared helper; glibc-2.41 / gcc-14 adopt the same helper in follow-ups (the expensive double-build rungs). Non-colliding: new file tests/repro-lib.sh + edits to tests/bootstrap-binutils-244-store-native.sh + its gate doc; no system/td.scm / check.sh / Makefile edit.
